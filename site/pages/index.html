Forerunner Documentation

Deterministic workflow execution for the JVM.

Forerunner is a small workflow engine built around explicit graph execution, immutable context flow, structured validation, and clear operational semantics. It stays close to the metal: no frameworks, no reflection, and no runtime magic.

Core Execution Model
Node → NodeOutcome → Workflow → WorkflowEngine → ExecutionResult

Workflows are directed graphs. Nodes transform context, the engine routes deterministically, and execution ends in a strongly typed result.

Design Objectives

Forerunner separates node behavior, graph topology, and execution policy into independent concerns. That keeps the runtime predictable and the surrounding tooling modular.

What Forerunner Is

Forerunner is designed for workflows that need clear control flow, explicit branching, validation accumulation, and reliable execution semantics. It is intentionally small. The goal is not to hide behavior. The goal is to make workflow behavior obvious.

Immutable Context Propagation Context moves through the graph as a value. Nodes read it, transform it, and return the next context state explicitly.
Explicit Routing `Continue`, `Next`, `Stop`, and `Fatal` make control flow visible both in code and in the editor.
Structured Violations Validation issues accumulate as `Violation` values with `WARNING` or `ERROR` severity instead of disappearing into side effects.
Deterministic Execution The engine walks the graph predictably. There is no reflection-based dispatch, hidden container state, or implicit runtime routing.

Repository Modules

The repository is a small ecosystem around the core engine: a JVM library, a TOML definition layer, future validation and CLI tooling, and a separate web editor.

core

The published library. It contains the workflow model, the Kotlin builder API, and the runtime engine, including `Node`, `Workflow`, `FlowBuilder`, `WorkflowEngine`, `Violation`, and `ExecutionResult`.

Published artifact: biz.digitalindustry.workflow:forerunner

dsl-toml

The canonical text-based workflow definition layer. This module owns the TOML schema and the mapping/parsing support around it.

validator

Reserved Kotlin/JVM module for future structural and semantic validation. One intended use case is CLI-driven validation of workflow definitions.

cli

Reserved command-line tooling module for inspection, validation, and execution-oriented workflow utilities.

editor

A web-based workflow editor built as a separate Node/Vite project. Atomica powers the UI/state layer, while Skeleton CSS and Skeleton-plus provide layout and styling.

examples/workflows

Example TOML workflow definitions used as references for schema evolution, editor behavior, and future tooling.

Execution Semantics

Core Runtime

A workflow is a directed graph with a `startNode`, a node map, and default `continueTo` routing inside the runtime model. Nodes return a `NodeOutcome` that determines how execution proceeds.

  • Continue follows default continue routing.
  • Next jumps directly to a named node.
  • Stop completes execution successfully.
  • Fatal terminates execution with an error.

Validation and Results

Violations accumulate as execution proceeds. The engine can either carry them through to completion or stop early when configured with fail-fast behavior.

  • Completed returns final context and violations.
  • ValidationFailed returns context plus accumulated violations.
  • Fatal returns context, error, and collected violations.
val engine = WorkflowEngine() val result = engine.execute(workflow, initialContext)

TOML Workflow Definitions

Forerunner also has a canonical TOML DSL for text-based workflow definitions. The text schema intentionally uses continue for fallback routing and routing for named branch routing.

workflow = "policy-underwriting" startNode = "validate" [nodes.validate] label = "Validate Policy" handler = "tenant.policy.validate" [nodes.finalize] type = "end" label = "Finalize" [continue] validate = "finalize" [routing.validate.highRisk] to = "manualReview"

Web Editor

The editor is a separate web tool for visually building workflows and editing the TOML side-by-side. It is intended to make graph structure, routing, and configuration easier to inspect without changing the underlying model.

Visual graph editing

Create nodes, route between them, rearrange graph layout, and inspect node or route details in a dedicated side inspector.

TOML-aware workflow editing

Edit workflow TOML directly and keep the visual representation aligned with the canonical text schema.

Implementation stack

Atomica provides the UI/state model. Skeleton CSS and Skeleton-plus define the visual shell and component styling.
